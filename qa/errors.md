### F004 · cli — new --kind <kind> --slug <slug> [--title <title>]

### F005 · cli — render [<slug>|--all]
- **Expected:** No args defaults to --all (discovers all .doc.md under sourcesDir + PRD subdirs); --all discovers the same; explicit paths render only those. Validates frontmatter before rendering; die() on any validation error. Output: {prdsDir}/{slug}/index.html (PRDs) or {plansDir}/{slug}.html (reports/working-docs) via outputPathFor(). Reports `rendered <src> -> <out>`. [code: src/doc-kit.mjs:1754-1768]
- **Found:** When an absolute path is passed as an explicit render argument, the engine crashes with an unhandled Node.js exception (full stack trace to stderr) rather than a clean error message. Root cause: parseSource() uses path.join(ROOT, srcPath) which does not treat an absolute srcPath as absolute — instead it prepends ROOT to the entire string, doubling the path. The spec says 'explicit paths render only those' without restricting to relative paths, and agents naturally use absolute paths.

### F125 · components — Resource tree (:::resources with linked rows + child lists)
- **Expected:** renderResources: a row with an href renders the label as an <a>; an unlinked parent followed by indented (>=2 space) children renders the children joined by " · " as links; a bare label renders plain text. All in <div class="rows">. [code: src/doc-kit.mjs:875-903; components.md:156-159]
- **Found:** The feature spec says 'a bare label renders plain text' — this is correct for 'Label ::' (has :: but empty href and no children), which renders the label text as the value. HOWEVER: a truly bare line with no :: separator at all is SILENTLY SKIPPED (splitRow returns null → continue). When such a bare line (e.g. 'Related Docs') precedes indented children, the children silently attach to the PREVIOUS parent; if that parent has an href, the children are then silently lost in rendering (since renderResources only renders href OR children, not both). A bare label at the start of a block with children causes the first child to become a top-level row and subsequent children to be silently dropped. No error is emitted. The stated behaviors (linked row, unlinked parent with ::, bare label with ::) all work correctly.
